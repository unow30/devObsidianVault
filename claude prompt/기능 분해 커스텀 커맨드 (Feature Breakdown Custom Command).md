## 1. 기능 분석

- **문제와 목표**: 이 기능이 해결하려는 문제와 기대 결과  
- **범위**: 이 기능에 *포함되는 것과 제외되는 것*  
- **주요 사용자 시나리오**: 주요 사용 케이스 2~3개  

### 기능 요구사항

- **UI/화면 요소**: UI 컴포넌트, 페이지, 상태 관리  
- **기술적 요구사항**: API 엔드포인트, 데이터베이스, 비즈니스 로직  
- **외부 연동**: 필요한 외부 서비스나 라이브러리  

---

## 2. 작업 분해

각 작업을 다음 형식으로 나열:

- **"무엇을" 하는 작업명**  
- **설명**: 구체적인 작업 내용  
- **예상 시간**: 1~4시간 단위  
- **선행 작업**: 선행 작업이 있다면 명시  
- **완료 조건**: 언제 완료로 볼 수 있는지  
- **위험 요소**: 중/소형의 주요 리스크  

---

## 3. 실행 관점 정리

### 병렬 처리 가능 여부

- **병렬 처리 가능**: 동시에 진행할 수 있는 작업들  
- **순차 처리 필요**: 반드시 순서대로 해야 하는 작업들  
- **우선순위 작업**: 리스크가 높거나 먼저 해결해야 할 작업들  

### 테스트 계획

- **기능 테스트**: 각 기능별 테스트 방법  
- **통합 테스트**: 전체 흐름을 검증하는 방법  
- **사용자 관점 테스트**: 사용자의 관점에서의 검증 포인트  

---

## 4. 산출물 및 체크포인트

- **마일스톤**: 주요 중간 목표들  
- **체크포인트**: 각 단계에서 확인할 사항과 대응 방안  
- **문서화 항목**: 문서로 정리되어야 하는 지점들  
- **모니터링**: 배포 후 확인할 지표들  

---

## 5. 추가 원칙/조건

- 각 작업은 1~4시간 내에 완료 가능한 크기로 분해  
- 개발 난이도와 비즈니스 가치를 고려한 우선순위 설정  
- 잠재적인 일정/품질 리스크 고려  
- 리서치가 필요한 작업은 조사/연구 작업으로 별도 분리
