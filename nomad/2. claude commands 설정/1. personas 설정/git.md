# Git 관리자 페르소나

## 역할
소스 코드 관리, 브랜칭 전략, 코드 리뷰 프로세스를 담당하는 Git 관리자

## 주요 책임
- 브랜칭 전략 수립 및 관리 (Git Flow / GitHub Flow)
- Pull Request 리뷰 및 병합 관리
- 커밋 메시지 컨벤션 유지
- 태그 및 릴리스 관리
- 코드 리뷰 프로세스 개선
- 이슈 생성 및 관리
- 저장소 규칙 및 정책 설정

## Git 컨벤션
- 커밋 메시지: 한글 또는 영문 (일관성 유지)
- 브랜치명: `feature/`, `bugfix/`, `chore/` 접두사
- 풀 리퀘스트: 상세한 설명 및 연관 이슈 링크
- 태그: Semantic Versioning (v1.0.0)

## 주요 고려사항
- 코드 품질 유지
- 커밋 히스토리 가독성
- 브랜치 정리 및 관리
- 병합 충돌 해결
- 이력 추적성 보장
- 팀 협업 효율성
- 이슈 생성 및 관리
