## 완료된 작업 요약
### Phase 1: 데이터베이스 스키마 설계
다음 테이블들을 생성했습니다:
  1. city_metrics - 실시간 메트릭 데이터
    - 현재 기온, AQI, 인터넷 속도, 월 생활비 등
    - 각 도시마다 최신 메트릭만 저장 (UNIQUE 제약)
  2. user_ratings - 사용자 평가
    - 사용자별 도시 평가 (좋아요/싫어요/중립)
    - 각 사용자는 도시당 하나의 평가만 가능
  3. user_reviews - 상세 리뷰
    - 제목, 내용, 별점 포함
    - 사용자가 여러 리뷰 작성 가능
  4. users - 사용자 프로필 (Supabase Auth 확장)
    - username, profile_image, bio 등

## Phase 2: Row Level Security (RLS) 정책
RLS 정책이 보안 베스트 프랙티스를 준수하며 잘 정리되어 있습니다. city_metrics는 읽기 전용 public 접근, 사용자 데이터는 인증 사용자만 조작 가능하도록 설계되었습니다.

| 테이블              | SELECT | INSERT | UPDATE     | DELETE     |
| ---------------- | ------ | ------ | ---------- | ---------- |
| **city_metrics** | Public | ❌      | ❌          | ❌          |
| **user_ratings** | Public | Auth   | Auth (자신만) | Auth (자신만) |
| **user_reviews** | Public | Auth   | Auth (자신만) | Auth (자신만) |
| **users**        | Public | ❌      | Auth (자신만) | ❌          |
