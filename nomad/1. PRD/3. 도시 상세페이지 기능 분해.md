
  1. 기능 분석

  문제와 목표

  - 문제: 홈페이지의 도시 카드에서는 제한된 정보(종합점수, 생활비, 인터넷, 날씨 등)만 확인 가능. 사용자가 특정 도시에 대한 상세한 정보를 얻기 위해서는 별도 페이지가 필요
  - 목표: 각 도시에 대한 종합적이고 심층적인 정보를 제공하는 상세 페이지를 구축하여, 디지털 노마드가 도시 선택에 필요한 모든 정보를 한 곳에서 확인할 수 있도록 함

  범위

  포함되는 것:
  - 도시 개요 및 히어로 섹션 (대표 이미지, 종합점수, 핵심 지표)
  - 상세 지표 섹션 (생활비, 인터넷, 날씨, 미세먼지, 작업 공간, 교통 등)
  - 사용자 리뷰 및 평가 시스템
  - 지도 및 위치 정보
  - 추천 작업 공간 목록
  - 관련 도시 추천
  - SEO 최적화 및 메타 태그

  제외되는 것:
  - 예약 시스템 (숙소, 작업 공간)
  - 커뮤니티 포럼 기능
  - 실시간 채팅
  - 사용자 간 메시징

  주요 사용자 시나리오

  1. 도시 정보 탐색: 사용자가 홈페이지에서 특정 도시 카드를 클릭 → 상세 페이지로 이동 → 생활비, 날씨, 작업 환경 등을 종합적으로 확인
  2. 리뷰 확인 및 작성: 사용자가 도시 상세 페이지에서 다른 노마드들의 리뷰를 확인 → 본인의 경험을 공유하기 위해 리뷰 작성
  3. 작업 공간 검색: 사용자가 도시 상세 페이지에서 추천 카페/코워킹 스페이스 목록 확인 → 지도에서 위치 확인

  기능 요구사항

  UI/화면 요소

  - 히어로 섹션 (도시 이름, 대표 이미지, 종합점수)
  - 핵심 지표 카드 그리드 (생활비, 월세, 인터넷, 날씨, 미세먼지, 교통 등)
  - 상세 정보 탭/섹션 (개요, 생활, 작업, 날씨, 교통)
  - 리뷰 섹션 (평가 분포, 리뷰 목록, 리뷰 작성 폼)
  - 지도 컴포넌트 (카카오맵 API)
  - 작업 공간 목록 컴포넌트
  - 관련 도시 추천 캐러셀
  - 공유 버튼 (SNS 공유)
  - 반응형 레이아웃 (모바일/태블릿/데스크톱)

  기술적 요구사항

  - API 엔드포인트:
    - GET /api/cities/:cityId - 도시 상세 정보
    - GET /api/cities/:cityId/reviews - 리뷰 목록
    - POST /api/cities/:cityId/reviews - 리뷰 작성
    - GET /api/cities/:cityId/workspaces - 작업 공간 목록
    - GET /api/cities/:cityId/weather-history - 날씨 히스토리
  - 데이터베이스 스키마:
    - cities 테이블 (기본 정보, 좌표, 설명 등)
    - city_metrics 테이블 (생활비, 인터넷 속도 등 지표)
    - reviews 테이블 (사용자 리뷰, 평점)
    - workspaces 테이블 (카페/코워킹 스페이스 정보)
  - 상태 관리: 도시 데이터, 리뷰, 필터 상태
  - 데이터 페칭: React Query를 사용한 캐싱 및 무한 스크롤 (리뷰)

  외부 연동

  - 기상청 API: 실시간 날씨 및 히스토리 데이터
  - 에어코리아 API: 미세먼지 데이터
  - 카카오맵 API: 지도 표시, 작업 공간 위치 마커
  - 카카오 로컬 API: 카페/작업 공간 검색
  - Open Graph: SNS 공유 시 메타 태그
