
현재 웹사이트를 개선하기 위해서 실행 계획을 제작한다.
여러 단계에 걸쳐서 계획을 구현하고 각 단계는 Phase라고 부른다. Phase1, Phase2 등
각 단계는 그 자체적으로 실행이 가능한 형태로 완성되야 한다.
점진적으로 프로젝트를 개선해나가야 한다.
각 단계별로 독립적인 에이전트가 작업할 수 있도록 충분한 문맥을 작성해라. 최소 아래 내용을 포함한다.
- 이번 단계에서 개선할 사항 오버뷰
- 수정 개선해야 하는 기능 체크박스로 나열
- 작업 완료 후 검증, 확인 기능들을 체크박스로 나열
- 아직 데이터베이스는 사용하지 않을거고 가짜 데이터로 진행할 것이다.
- 기능을 구현하지 말고 단계별 계획을 출력해라

Phase1
별점 평점 삭제하고 '좋아요', '싫어요' 버튼만 남기기. 버튼을 눌렀을 때 좋아요 상태에 따라 아이콘 색상 변경하고, 좋아요, 싫어요 숫자 변경하기

Phase2
네비게이션에 다른 페이지로 이동하는 버튼 삭제

Phase3
필터 정보는 다음과 같이 설정
- 예산: 100만원 이하, 100~200만원, 200만원 이상
- 지역: 전체, 수도권, 경상도, 전라도, 강원도, 제주도, 충청도
- 환경: 자연친화, 도심선호, 카페작업, 코워킹 필수
- 최고계절: 봄, 여름, 가을 ,겨울

Phase4
카드 구조 변경하기. 위에서 정의한 필터에 해당하는 예산, 지역, 환경, 최고 계절을 key, value로 보여주기

Phase5
가짜 데이터 수정하기. 각 도시별로 필터에 해당되는 요소 최소 하나씩 추가하기

Phase6
로그인 성공 시 로그인, 회원가입 버튼 위치에 버튼 대신 로그인 유저의 email 정보를 우측 상단에 표시한다.
email을 클릭하면 프로필 버튼, 로그아웃 버튼이 토글된다.
